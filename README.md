# Hannah Szu-Han Wang — Personal Academic Website

Built with [Quarto](https://quarto.org/) and deployed to [GitHub Pages](https://szuhannah.github.io/).

## Prerequisites

1. **Install Quarto CLI**: <https://quarto.org/docs/get-started/>
2. **Install R** (if you want to run R code in blog posts): <https://cran.r-project.org/>
3. **VSCode extensions**:
   - [Quarto](https://marketplace.visualstudio.com/items?itemName=quarto.quarto) (required)
   - [R](https://marketplace.visualstudio.com/items?itemName=REditorSupport.r) (optional, for R code execution)

## First-Time Setup (Migrating from Old Site)

**⚠️ IMPORTANT: You must download your hosted PDFs before overwriting the repo!**

### Step 1: Clone your existing repo

```bash
git clone https://github.com/SzuHannah/SzuHannah.github.io.git
cd SzuHannah.github.io
```

### Step 2: Download all self-hosted files

```bash
bash download_files.sh
```

This downloads 7 PDFs + your profile photo from the live site into `files/` and `images/`.
Verify the files are there:

```bash
ls -la files/posters/ files/slides/ files/papers/ images/
```

### Step 3: Remove old Hugo/blogdown files (keep .git/ and downloaded files)

```bash
# Move downloaded files to a temp location
mv files /tmp/site-files
mv images /tmp/site-images

# Delete everything except .git
find . -maxdepth 1 ! -name '.git' ! -name '.' -exec rm -rf {} +

# Copy in the new Quarto project files
# (extract quarto-site.zip contents here, or copy files manually)

# Restore downloaded files
mv /tmp/site-files files
mv /tmp/site-images images
```

### Step 4: Render and push

```bash
quarto render
git add -A
git commit -m "migrate to Quarto"
git push
```

### Step 5: Configure GitHub Pages (one-time)

Go to repo **Settings → Pages** → set:
- Source: **Deploy from a branch**
- Branch: **main**
- Folder: **/docs**

Your site will be live at https://szuhannah.github.io/ within ~2 minutes.

## Local Development

```bash
# Preview with live reload (opens browser automatically)
quarto preview

# Full render (builds into docs/)
quarto render
```

## How to Update Content

### Edit existing pages
Edit the `.qmd` file and save. If `quarto preview` is running, changes appear instantly.

### Add a new blog post
1. Create a new folder: `blog/posts/YYYY-MM-DD-short-title/`
2. Create `index.qmd` inside it:
   ```yaml
   ---
   title: "Your Post Title"
   description: "A brief description."
   date: YYYY-MM-DD
   categories: [R, Data Science]
   ---
   ```
3. Write content below the YAML (supports R code chunks)
4. The blog listing page auto-discovers new posts

### Add a new publication / project / talk
Open the corresponding `.qmd` file and copy an existing `<div class="card-item">` block, then edit.

### Deploy changes

```bash
quarto render
git add -A
git commit -m "describe your changes"
git push
```
